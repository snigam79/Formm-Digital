The Build
Exact database schema, scoring engine, percentile system, API surface, and share card generator. Everything needed to make Formm Score real and defensible.
Database — add these tables today
Every analysis you run without these tables is data you lose permanently. This is the single most urgent action in this entire document.
rep_data — the core asset
One row per rep per analysis. Raw joint angles. Never delete this. This is the moat.
CREATE TABLE rep_data ( id UUID PRIMARY KEY, analysis_id UUID REFERENCES analyses, rep_number INT, knee_flexion_deg FLOAT, hip_flexion_deg FLOAT, ankle_dorsiflexion_deg FLOAT, valgus_angle_deg FLOAT, trunk_angle_deg FLOAT, symmetry_ratio FLOAT, rep_score INT, captured_at TIMESTAMPTZ );
metric_scores + formm_score
Five rows per analysis — one per dimension. Benchmarks stored alongside for auditability.
CREATE TABLE metric_scores ( id UUID PRIMARY KEY, analysis_id UUID REFERENCES analyses, metric_name VARCHAR(40), value FLOAT, benchmark_min FLOAT, benchmark_max FLOAT, score_0_100 INT, weight FLOAT ); CREATE TABLE formm_score ( user_id UUID PRIMARY KEY, overall_score INT, level VARCHAR, percentile_sport INT, analyses_count INT, last_updated TIMESTAMPTZ );
Scoring engine — TypeScript, server-side only
Weights are a trade secret. Never in the codebase. Environment variables only. Runs server-side after every POST /api/analyses.
// Loaded from env — never hardcoded in source const WEIGHTS = { form_accuracy: +process.env.W_FORM, // 0.35 range_of_motion: +process.env.W_ROM, // 0.20 stability: +process.env.W_STAB, // 0.20 symmetry: +process.env.W_SYM, // 0.15 improvement_rate: +process.env.W_IMP, // 0.10 }; function calcFormmScore(metrics: MetricScore[]): number { const weighted = metrics.reduce((sum, m) => sum + (m.score_0_100 * WEIGHTS[m.metric_name]), 0); const totalW = Object.values(WEIGHTS).reduce((a,b)=>a+b,0); return Math.round(weighted / totalW); } function assignLevel(score: number): string { if (score >= 85) return 'Elite / Pro'; if (score >= 75) return 'Collegiate'; if (score >= 60) return 'Advanced'; if (score >= 40) return 'Intermediate'; return 'Recreational'; } // Cross-movement composite — weighted by session count function crossMovementScore(movements: MovementScore[]): number { const totalSessions = movements.reduce((s,m)=>s+m.session_count,0); const weighted = movements.reduce((s,m)=>s+(m.score*m.session_count),0); return Math.round(weighted / totalSessions); }
Percentile engine — nightly cron at 02:00 IST
Populates score_distributions table. Lookup is O(1) — a single indexed query. Runs in ~200ms at 50k users.
-- Rebuild nightly. Runs in 200ms at 50k users. INSERT INTO score_distributions (sport, movement, p25, p50, p75, p90, updated_at) SELECT u.sport, a.movement, PERCENTILE_CONT(0.25) WITHIN GROUP (ORDER BY fs.overall_score) AS p25, PERCENTILE_CONT(0.50) WITHIN GROUP (ORDER BY fs.overall_score) AS p50, PERCENTILE_CONT(0.75) WITHIN GROUP (ORDER BY fs.overall_score) AS p75, PERCENTILE_CONT(0.90) WITHIN GROUP (ORDER BY fs.overall_score) AS p90, NOW() FROM formm_score fs JOIN users u ON fs.user_id = u.id JOIN analyses a ON a.user_id = fs.user_id WHERE fs.analyses_count >= 5 GROUP BY u.sport, a.movement ON CONFLICT (sport, movement) DO UPDATE SET p25=EXCLUDED.p25, p50=EXCLUDED.p50, p75=EXCLUDED.p75, updated_at=NOW();

The India-specific opportunity
Why India is the right first market
30,000+ cricket academies nationwide with no objective movement quality measurement. BCCI and state boards spend heavily on coaching but have no standardised biomechanical benchmarking tool. Formm Scout Score — validated against peer-reviewed research — is the first credible candidate. No Western competitor has Indian-specific population data.
Indian population calibration
The current benchmarks (Escamilla, Hewett, McGill) are from predominantly Western populations. An Indian-specific reference population makes the percentile layer more meaningful domestically — and impossible for a Western competitor to replicate without their own Indian dataset. This is a secondary moat that builds automatically as you grow.
The Science

5-dimension framework — sources and weights
DimensionWeightWhat it measuresPrimary sourceWhy this weight
Form accuracy35%Joint angles vs peer-reviewed ideal ranges. Knee valgus, hip depth, spinal alignment.Hewett et al. 2005 (AJSM) — valgus as ACL predictor. Escamilla et al. 2001 (JOSPT).Strongest injury predictor. Valgus moment predicts ACL at p<0.001.
Range of motion20%Depth achieved vs minimum required. Hip below parallel, ankle dorsiflexion available.Straub & Powers 2024 (IJSPT). Hartmann et al. 2013 (Sports Medicine).Mobility deficit is the 2nd most common cause of compensatory movement.
Stability20%Deviation from neutral planes. Lateral trunk lean, tibial sway, anterior pelvic tilt.McGill 2007 (Low Back Disorders). Cholewicki & McGill 1991 (J Biomechanics).Instability under load = primary low back injury mechanism.
Symmetry15%Left vs right side balance. Asymmetry >15% under load is a risk marker.Kiesel et al. 2007 — FMS asymmetry and injury. Hadzic et al. 2012.Asymmetry predicts overuse injury with OR ~1.7. Improves slowly — lower weight.
Improvement rate10%Score trajectory across sessions. Rewards consistent deliberate practice.Ericsson 1993 (deliberate practice). Composite of motor learning literature.Lowest weight — must not let one good session distort an otherwise weak score.

Weight calibration — Year 1 vs Year 2+
Year 1 — literature weights
Apply to all users regardless of sport. Based entirely on injury-prediction literature. MVP weighting.
form_accuracy: 35% // strongest predictor range_of_motion: 20% // mobility deficit stability: 20% // spine/joint safety symmetry: 15% // bilateral balance improvement_rate: 10% // consistency reward
Year 2+ — sport-adjusted weights
Once you have enough per-sport data, calibrate weights to sport-specific injury patterns.
// Cricket (batting/bowling) shoulder_symmetry: 25% // rotational sport trunk_stability: 25% form_accuracy: 25% range_of_motion: 15% improvement_rate: 10% // Running ankle_dorsiflexion: 30% // impact mechanics hip_alignment: 25% form_accuracy: 25% symmetry: 12% improvement_rate: 8%

Prospective cohort. 12-month follow-up. Primary outcome: injury incidence.

The Product
The 5 moments that determine whether users care about their score. Empty state design. Notification rules. What makes the share card get shared.
The 5 moments that determine retention
Empty state design — every zero has a job
Never show
"Your Formm Score: 0/100 · Top 99% of all athletes." This is what the current build shows. It communicates total failure to a user who has done nothing wrong.
Any bar chart flat at zero. Percentile shown when score is 0. "No data yet." "Complete your profile to see your score."
Always show
The lock screen with a progress bar. "1 of 5 analyses · 4 more needed · any movement counts." Turns incompleteness into a goal.
Ghost chart showing what the populated version will look like. The milestone you are working toward — not the empty state you are stuck in.

Share card — what makes it get shared
Must contain
Score (large, purple). Level badge. Percentile in the user's sport. Top 3 metric scores. Month/year. "Formm" brand prominent. QR code linking to formm.app (future).
Test at 375px width (smallest phone). The score must be readable as a thumbnail in an Instagram Story without the card being opened full-screen.
Why "Top 29%" gets shared and "72" doesn't
The number alone means nothing. The social context — "Top 29% for squat form in Gym / Lifting" — implies community, competition, and achievement simultaneously.
The share prompt should appear after the level-up animation completes — not before. The emotional high of the animation is the trigger. Asking before it lands is a cold ask.

Data Layer
Full database architecture with row-level security. Anonymisation pipeline for the validation study. DPDP Act 2023 compliance. Retention policy.
Database architecture with RLS
Schema separation
PII and movement data in separate schemas. They are never joined in the application layer — only via server-side functions with explicit access control.
-- Schema: public (movement data, no PII) analyses, rep_data, metric_scores, formm_score, score_distributions, score_history, milestones -- Schema: user (PII — strict RLS) users, user_profile, subscriptions -- Schema: b2b coaches, coach_athletes, club_dashboards -- Row-level security example CREATE POLICY "users own their data" ON rep_data FOR ALL USING ( analysis_id IN ( SELECT id FROM analyses WHERE user_id = auth.uid() ) );
Anonymisation pipeline
IRB requires anonymisation before transfer to university. Hashed user IDs. No PII in the export. Only movement and score data.
-- Export for university research partner SELECT encode(sha256(user_id::text::bytea), 'hex') AS anon_user_id, u.sport, u.training_level, u.age_bracket, -- 18-24, 25-34 etc. a.movement, a.overall_score, fs.overall_score AS formm_score, fs.level, -- NOT: name, email, device_id a.analysed_at::date AS analysis_date FROM analyses a JOIN formm_score fs ON fs.user_id = a.user_id JOIN users u ON u.id = a.user_id WHERE u.research_consent = true;
DPDP Act 2023 compliance — India
What the Act requires for camera/biometric data
Joint angle data from camera analysis is biometric data under DPDP Act 2023. Required before 10,000 users:
1. Consent notice — explicit, purpose-specific, plain language at signup 2. Data principal rights — implement: Right to access their data Right to correction Right to erasure (delete account) Right to grievance redressal 3. Data Fiduciary registration (required above threshold user count) 4. Research export: Separate research consent Anonymise before transfer Data Processing Agreement with university
Required privacy policy language
This exact language is required in your privacy policy for the camera data processing:
"Formm processes video from your device camera to analyse your movement form. VIDEO: Processed locally on your device using TensorFlow.js. Video is NOT uploaded to Formm servers. JOINT ANGLES: The numerical output of the analysis (e.g. knee flexion: 112°) is stored on our servers, associated with your account. RETENTION: Joint angle data is retained for the lifetime of your account and deleted within 30 days of your deletion request. PURPOSE: Used to calculate your Formm Score, provide coaching recommendations, and (with your separate consent) contribute to anonymised sports science research."

Go-to-Market
12-month phased roadmap. Four acquisition channels. The brand message that creates a new category.
12-month roadmap
Acquisition channels
Channel 1 — Cricket academy partnerships
Find 3–5 cricket academies in one city. Offer 90-day free access to the coach portal. Ask them to have all their athletes (15–30 per academy) create accounts and run their first form analyses.
This gets you: 50–150 real users with high-quality data, a case study, and a referral channel. The pitch to parents: "See a scientific score for your child's form." No other tool offers this.
Channel 2 — Gym chain white-label
Approach mid-market gyms (₹2,000–4,000/month — serious recreational athletes). Offer branded coach dashboard for their trainers. Revenue: ₹10/member/month paid by the gym.
500 members × ₹10 = ₹5,000/month per gym. 20 gym chains = ₹1L/month from partnerships. Drives Formm adoption among members who want to track their progress.
Channel 3 — Instagram Reels format
3 Reels per week. Format: before/after form check with Formm Score overlay. Specific fault identified, fix applied, score rechecked. Not generic fitness content — specific, scientific, Formm-branded.
Each Reel is both acquisition content and product education. "What does an 84/100 squat look like vs a 61/100 squat?" — this question is inherently shareable and demonstrates the product instantly.
Channel 4 — Share card flywheel
Every level-up generates a share card. "I scored 84/100 for squat form — top 18%" communicates more than "I completed a workout." It implies community, competition, and scientific credibility simultaneously.
The flywheel requires users to reach level-ups. That requires the lock screen to be compelling enough to drive 5 analyses. This is a product problem as much as an acquisition problem.
The brand message — one sentence to own
"You track your workouts. You track your sleep. You don't track whether you move correctly. Formm does."
This is not "AI-powered form correction" (feature). Not "personalised sports science" (category too broad). It is specifically the gap between tracking effort and tracking quality. That gap is real, universally felt by anyone who trains seriously, and Formm is the first product to close it.
The category Formm creates
"Movement quality" does not exist as a consumer category yet. Fitness apps track volume. Wearables track physiology. Nobody tracks whether you move well. Formm's job is to make "movement quality" a category people understand, care about, and measure.
The question Formm needs in common use
"What's your Formm Score?" — in the same way "What's your Strava FTP?" or "What's your WHOOP Recovery?" became shorthand in performance communities. The score is the brand, and the brand is the score.
